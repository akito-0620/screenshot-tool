# スクショを構造的に保存できるツール 設計書

## 1. 題名

**スクショを構造的に保存できるツール**

## 2. 目的

スクリーンショットを撮影した後に毎回保存先を探すのではなく、撮影前に保存先フォルダを明示的に設定しておくことで、講義・研究・就活・資料作成などの用途ごとにスクリーンショットを整理しやすくする。

例：

- 「木5の講義」のスクリーンショットは、事前に `木5` フォルダを選択しておき、自動的にそこへ保存する。
- 研究ミーティング用のスクリーンショットは、`研究/ミーティング` のようなフォルダに保存する。
- 新しい授業やプロジェクトが始まった場合は、アプリ上から保存先フォルダを新規作成できる。

## 3. 想定ユーザー

- 講義資料、研究資料、就活資料、Webページ、アプリ画面などを頻繁にスクリーンショットで保存するユーザー
- スクリーンショットがデスクトップやダウンロードフォルダに散らばることを避けたいユーザー
- 撮影前に保存先を決めて、後から整理する負担を減らしたいユーザー

## 4. 対応環境

まずは **Windowsデスクトップアプリ** として実装する。

推奨技術スタック：

- 言語：Python
- GUI：PyQt6
- スクリーンショット取得：mss または PyQt6 のスクリーン取得機能
- 画像保存：Pillow
- 設定保存：JSONファイル

## 5. 基本コンセプト

このアプリでは、保存先フォルダの設定を2段階に分ける。

### 5.1 第1段階：親フォルダの設定

親フォルダは、スクリーンショット保存先の基準となるフォルダである。

例：

```text
C:/Users/username/Pictures/Screenshots
C:/Users/username/Documents/LectureScreenshots
D:/Research/Captures
```

親フォルダは頻繁には変更しない。月に数回、または用途が大きく変わったときに変更する想定。

親フォルダの設定方法：

1. 「参照」ボタンからフォルダ選択ダイアログで指定する
2. パスをテキスト入力して指定する

指定された親フォルダは、アプリ終了後も設定ファイルに保存され、次回起動時に復元される。

### 5.2 第2段階：保存先フォルダの設定

保存先フォルダは、実際にスクリーンショット画像が保存されるフォルダである。

保存先フォルダは、親フォルダの中にある子フォルダから選ぶ。

例：

```text
親フォルダ: C:/Users/username/Documents/LectureScreenshots
保存先フォルダ: 木5
最終保存先: C:/Users/username/Documents/LectureScreenshots/木5
```

保存先フォルダは頻繁に変更する想定。講義、作業内容、調べ物のテーマごとに切り替える。

保存先フォルダの設定方法：

1. 親フォルダ内に存在するフォルダ一覧から選択する
2. 新規フォルダ名を入力し、親フォルダ内に作成する

## 6. 主な機能要件

### 6.1 親フォルダ設定機能

ユーザーはアプリ上で親フォルダを設定できる。

要件：

- 現在の親フォルダパスを表示する
- 「参照」ボタンでフォルダ選択ダイアログを開く
- テキストボックスにパスを直接入力できる
- 入力されたパスが存在するか検証する
- 存在しないパスの場合はエラーを表示する
- 設定された親フォルダをJSONに保存する
- 親フォルダ変更時に、第2段階のフォルダ一覧を更新する

### 6.2 保存先フォルダ選択機能

ユーザーは親フォルダ内の子フォルダから、スクリーンショットの保存先を選択できる。

要件：

- 親フォルダ内のフォルダ一覧を表示する
- フォルダ一覧はコンボボックスまたはリストで表示する
- 保存先フォルダを選択すると、現在の保存先として反映される
- 現在の保存先フォルダ名とフルパスを表示する
- 親フォルダ内に子フォルダが存在しない場合は、「フォルダを作成してください」と表示する

### 6.3 新規保存先フォルダ作成機能

ユーザーはアプリ上から、親フォルダ内に新しい保存先フォルダを作成できる。

要件：

- 新規フォルダ名を入力するテキストボックスを用意する
- 「作成」ボタンで親フォルダ内にフォルダを作成する
- 作成後、フォルダ一覧を自動更新する
- 作成したフォルダを現在の保存先として自動選択する
- 同名フォルダが既に存在する場合はエラーではなく、そのフォルダを選択するか、確認メッセージを表示する
- Windowsで使用できない文字を含むフォルダ名は拒否する

禁止文字の例：

```text
< > : " / \\ | ? *
```

### 6.4 範囲選択スクリーンショット機能

ユーザーはマウスで画面上の範囲を選択し、その範囲だけをスクリーンショットとして保存できる。

要件：

- 「範囲を選択して撮影」ボタンを押すと、画面全体に半透明のオーバーレイを表示する
- ユーザーがマウスでドラッグした矩形範囲を取得する
- ドラッグ終了時に、その範囲のスクリーンショットを撮影する
- 撮影した画像を現在の保存先フォルダにPNG形式で保存する
- 保存後、保存完了メッセージを表示する
- Escキーで範囲選択をキャンセルできる

### 6.5 ファイル名自動生成機能

スクリーンショット保存時にファイル名を自動生成する。

ファイル名形式：

```text
screenshot_YYYYMMDD_HHMMSS.png
```

例：

```text
screenshot_20260516_131245.png
```

同じ秒に複数保存された場合は連番を付ける。

例：

```text
screenshot_20260516_131245.png
screenshot_20260516_131245_01.png
screenshot_20260516_131245_02.png
```

### 6.6 設定保存機能

アプリの設定をJSONファイルに保存する。

保存する情報：

```json
{
  "parent_folder": "C:/Users/username/Documents/LectureScreenshots",
  "selected_child_folder": "木5"
}
```

設定ファイルの保存場所：

```text
アプリ実行ディレクトリ/config.json
```

または、将来的には以下のようなユーザー設定ディレクトリに保存してもよい。

```text
C:/Users/username/AppData/Roaming/StructuredScreenshotTool/config.json
```

## 8. 処理フロー

### 8.1 アプリ起動時

1. `config.json` を読み込む
2. 設定ファイルが存在しない場合は初期状態で起動する
3. 親フォルダが設定済みか確認する
4. 親フォルダが存在する場合、子フォルダ一覧を読み込む
5. 前回選択していた子フォルダが存在する場合、それを選択する
6. 存在しない場合は未選択状態にする

### 8.2 親フォルダ変更時

1. ユーザーが参照またはパス入力で親フォルダを指定する
2. 指定されたパスが存在するか検証する
3. 存在する場合、親フォルダとして保存する
4. 子フォルダ一覧を再読み込みする
5. 以前の保存先フォルダ選択をリセットする
6. 画面表示を更新する

### 8.3 子フォルダ選択時

1. ユーザーがフォルダ一覧から子フォルダを選択する
2. `parent_folder / selected_child_folder` を現在の保存先として設定する
3. 設定を `config.json` に保存する
4. 現在の保存先表示を更新する

### 8.4 新規フォルダ作成時

1. ユーザーが新規フォルダ名を入力する
2. フォルダ名が空でないか確認する
3. 禁止文字が含まれていないか確認する
4. 親フォルダが設定されているか確認する
5. 親フォルダ内に新規フォルダを作成する
6. フォルダ一覧を更新する
7. 作成したフォルダを現在の保存先として選択する
8. 設定を保存する

### 8.5 スクリーンショット撮影時

1. 現在の保存先フォルダが設定されているか確認する
2. 保存先フォルダが存在するか確認する
3. メイン画面を一時的に隠す、または最小化する
4. 画面全体に範囲選択用オーバーレイを表示する
5. ユーザーがマウスで範囲をドラッグする
6. 選択範囲の座標を取得する
7. 選択範囲のスクリーンショットを撮影する
8. 自動生成したファイル名でPNG保存する
9. メイン画面を再表示する
10. 保存完了メッセージを表示する

## 11. 実装方針

### 11.1 メインウィンドウ

PyQt6 の `QMainWindow` または `QWidget` で実装する。

主な構成：

- 親フォルダ設定エリア
- 保存先フォルダ設定エリア
- 撮影ボタンエリア
- ステータス表示エリア

### 11.2 範囲選択オーバーレイ

`QWidget` をフルスクリーン表示し、半透明背景にする。

実装方針：

- `setWindowFlags(Qt.FramelessWindowHint | Qt.WindowStaysOnTopHint)` を使用する
- `setWindowOpacity` または半透明描画で背景を暗くする
- `mousePressEvent` で始点を記録
- `mouseMoveEvent` で選択中の矩形を更新
- `mouseReleaseEvent` で終点を確定
- `paintEvent` で選択矩形を描画
- `keyPressEvent` で Esc キャンセルを処理

### 11.3 スクリーンショット撮影

範囲選択で取得した座標をもとに、指定範囲のスクリーンショットを保存する。

実装候補：

#### 候補A：mssを使う

メリット：

- 高速
- 複数モニター対応を拡張しやすい
- Pythonで扱いやすい

#### 候補B：PyQt6のscreen grabを使う

メリット：

- PyQt6のみで実装できる
- 依存ライブラリが少ない

最初の実装では、安定性を重視して `mss + Pillow` を推奨する。

## 12. バリデーション

### 12.1 親フォルダ

- 空文字は不可
- 存在しないパスは不可
- ファイルパスは不可
- 読み取り権限がない場合は不可

### 12.2 子フォルダ

- 空文字は不可
- Windows禁止文字を含む名前は不可
- 前後の空白は削除する
- `.` や `..` のみの名前は不可
- 既存フォルダ名と同じ場合は新規作成せず、そのフォルダを選択する

### 12.3 スクリーンショット範囲

- 幅または高さが0の範囲は保存しない
- 極端に小さい範囲の場合は警告してもよい
- Esc押下時は保存しない

## 16. 将来的な拡張案

### 16.1 ホットキー対応

アプリを常駐させ、ショートカットキーで範囲選択スクリーンショットを開始できるようにする。

例：

```text
Ctrl + Shift + S
```

### 16.2 最近使った保存先

最近使った子フォルダを上位に表示する。

### 16.3 用途別プリセット

親フォルダと子フォルダの組み合わせをプリセットとして保存する。

例：

- 講義
- 研究
- 就活
- 趣味
- 一時保存

### 16.4 自動タグ付け

保存時に、フォルダ名や日時をもとにメタ情報を付与する。

### 16.5 OCR連携

スクリーンショット内の文字をOCRで読み取り、画像と一緒にテキストファイルとして保存する。

### 16.6 保存後の通知

Windows通知で保存先とファイル名を表示する。

### 16.7 保存先を開くボタン

現在の保存先フォルダをエクスプローラーで開けるボタンを追加する。
